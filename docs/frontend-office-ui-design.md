# AlphaOS 前端设计文档：AI 投研办公室（Pixel Office UI）

版本：v0.1（待评审）  
分支：`codex/frontend-office-ui`  
依据素材：5 张产品截图（投研大厅 / 多 Agent 作战室 / 专家中心 / 研究报告追问 / Manager 任务澄清）

---

## 1. 目标与范围

把 AlphaOS 后端（Manager → 动态 DAG → 专家池 → ResultAggregator）包装成一个
「像素风 AI 投研办公室」多页面前端。核心体验：

1. **首次进入是 Manager（投资经理）的一对一聊天**，社交软件聊天框风格；
   聊天收尾（确认并启动 / 跳过）后进入主页「投研大厅」。
2. **投研大厅**：任务入口 + 团队概览 + 办公室实况预览。
3. **多 Agent 作战室**：像素小人在办公室里真实走动、对话、围圆桌开会；
   被调用的 Agent 输出文字；Agent 间交流会走到对方工位；点击小人可
   启用/停用该专家（停用后不再被任务调用）。
4. **专家中心**：6 位专家的档案、能力雷达、技能列表、近期任务。
5. **研究报告**：报告详情 + 与 Manager 的继续追问对话。

**铁律（沿用 AGENTS.md）**：每个按钮都必须有真实交互行为；前端只渲染契约
与证据，不编造研究结论；演示数据必须标注 `DEMO`。

## 2. 技术方案

| 决策 | 方案 | 理由 |
| --- | --- | --- |
| 技术栈 | 零构建：原生 HTML + ES Modules + 单套 CSS 设计系统 | 与仓库现状一致（现有 `frontend/` 即零构建），符合「避免不必要依赖」 |
| 应用形态 | 单页壳 + Hash 路由（`#/chat`、`#/hall`、`#/warroom`、`#/experts`、`#/reports`） | 侧栏/顶栏不刷新；作战室 canvas 动画跨页面保持；无需框架 |
| 托管 | FastAPI `/static/` 静态托管（沿用 `backend/main.py` 现状） | 无需新增服务 |
| 像素场景 | Canvas 2D：静态办公室背景（可 imagegen 生成一张等距像素办公室底图）+ 程序化绘制的像素小人（行走/坐下/说话状态机） | 小人需要实时寻路行走与点击拾取，程序化渲染可控性最高 |
| 图表 | 轻量自绘 canvas 图表（折线/条形/雷达），不引入图表库 | 页面图表样式固定，自绘约 200 行，零依赖 |
| 数据模式 | **Demo（默认）**：本地剧本化事件流驱动全部动画与日志；**Live**：调 `/api/tasks` 等真实接口 | 与现有前端双模式一致；demo 不消耗 Ark/PandaData 配额 |
| 持久化 | `localStorage`：聊天完成标记、专家启用状态、模式开关、demo 历史任务 | 无后端存储需求 |

## 3. 信息架构与页面流转

```text
首次访问（localStorage 无标记）
  └─ #/chat  Manager 澄清聊天 ──「确认并启动」── 创建任务 ──► #/hall（大厅）
             │                        （任务进入执行中，办公室实况可见）
             └─「先去大厅逛逛」──────────────────────────────► #/hall
非首次访问 → 直接落 #/hall

#/hall 输入任务 →「启动 AI 投研团队」→ #/chat（带着该问题澄清）→「确认并启动」
  → #/warroom（实时执行）→ 完成 → #/reports（报告 + 追问）
```

## 4. 全局设计系统

- **配色**（取自截图）：背景 `#0a1628` / 面板 `#101f33` / 描边 `#1d3a5f`；
  主青 `#22d3ee`；成功绿 `#34d399`；警示黄 `#fbbf24`；危险红 `#f87171`；
  文字主 `#e8f1ff` / 次 `#7fa3c7`。
- **风格**：深色玻璃拟态卡片 + 像素描边高光（2px 硬阴影）；圆角 10px；
  标题带左侧 ✦ 装饰；像素元素只出现在角色、办公室场景、图标点缀。
- **字体**：系统栈 + `"Microsoft YaHei"`；数字用 `tabular-nums`。
- **布局骨架**：左侧 220px 侧栏；顶部 56px 状态栏；底部 40px 标语栏；
  内容区随路由切换。

## 7. 像素引擎技术要点

- **坐标系**：斜 45° 等距网格；工位 ×6、中央圆桌、大屏、白板、休息区沙发
  均有固定格位与碰撞格；走道预生成路点图，A* 寻路。
- **角色状态机**：`sit / idle / walk / talk / meeting / off`；行走 4 帧循环
  （左右脚交替 + 1px 起伏）；说话 = 站立 + 气泡 DOM 悬浮（跟随角色坐标）。
- **渲染循环**：`requestAnimationFrame`，逻辑 30 tick/s 与绘制解耦；
  页面隐藏时暂停（`visibilitychange`）。
- **点击拾取**：canvas 点击坐标 → 世界坐标 → 角色包围盒命中 → 操作卡。
- **可访问性**：canvas 旁配隐藏列表镜像当前各 Agent 状态，供读屏。

| 界面数据 | Demo 来源 | Live 来源 |

## 9. 验收标准
